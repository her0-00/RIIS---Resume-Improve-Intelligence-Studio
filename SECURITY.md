# Security Policy

## Supported Versions

Only the latest release of **IRIS** is actively supported for security updates. If you discover a vulnerability, please ensure you are testing against the main branch or the latest stable release tag.

| Version | Supported |
| ------- | --------- |
| v4.x    | ✅ Yes     |
| < v4.0  | ❌ No      |

## Reporting a Vulnerability

We take the security of our users and data extremely seriously. Since IRIS handles sensitive personal data (resumes, CV contents, contact details, profile pictures), we want to make sure all data processing remains secure and private.

> [!WARNING]
> Please do NOT report security vulnerabilities via public GitHub issues.

If you believe you have found a security vulnerability in this project, please report it responsibly by following these steps:

1. **Email the Maintainers**: Send a detailed email to the maintainers (you can specify your security email or use GitHub's private vulnerability reporting feature).
2. **Describe the Issue**: Provide a clear description of the vulnerability, including:
   - The affected component (e.g., PDF generation, Autonomous Hunter, file uploading).
   - Step-by-step instructions to reproduce the issue.
   - Any proof of concept (PoC) code or payloads used.
   - The potential impact of the vulnerability.

We will review all reports promptly and aim to acknowledge receipt of your report within 48 hours.

## Disclosure Policy

When a vulnerability is reported:
1. We will work to verify and patch the issue as quickly as possible.
2. A draft security advisory will be created.
3. Once the patch is tested and released, we will publish the advisory to inform the community, giving full credit to the reporter (unless they prefer to remain anonymous).

Thank you for helping keep IRIS and its community safe!
